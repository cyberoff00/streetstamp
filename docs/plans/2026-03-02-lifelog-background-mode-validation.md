# Lifelog 后台记录模式真机验收脚本

## 前置条件
- 机型：同一台 iPhone（避免设备差异）
- 系统：同一 iOS 版本
- 权限：定位 = 始终允许 + 精确定位
- Lifelog 开关：开启
- 省电模式：关闭
- 电量：两轮开始时尽量接近（例如都在 80% 左右）
- 网络：可用（但不要求持续在线）

## 路线要求（两轮一致）
- 时长：30-60 分钟
- 场景：有拐弯、地铁口/高楼遮挡、短暂停留
- 操作：启动后退到后台或锁屏，不手动划掉 App

## Round A：高精度
1. 进入设置，切换 `后台记录模式 = 高精度`。
2. 回到首页，确认未开启 Journey（只测被动 Lifelog）。
3. 按既定路线行走 30-60 分钟（可锁屏）。
4. 结束后打开 App，进入 Lifelog + Globe。
5. 记录以下指标。

## Round B：低精度
1. 进入设置，切换 `后台记录模式 = 低精度`。
2. 同样确认未开启 Journey。
3. 同一路线、同样时长再走一轮。
4. 结束后打开 App，进入 Lifelog + Globe。
5. 记录相同指标。

| 指标 | 高精度 | 低精度 | 结论 |
|---|---:|---:|---|
| 总时长（分钟） |  |  |  |
| 电量下降（%） |  |  |  |
| Lifelog 轨迹总长度（km） |  |  |  |
| 明显断线次数（主观计数） |  |  |  |

## 判定标准（你这次目标）
- 高精度相较低精度：
  - 轨迹连续性明显更好（断线更少、转弯更完整）。
  - 电量消耗更高但可接受（目标额外约 8%-15%/天量级）。
- 低精度：
  - 轨迹稀疏可接受。
  - 省电优势明显。

## 回归检查（必须）
- 开启 Journey（sport/daily）各跑 3-5 分钟：
  - 行为应与改动前一致。
  - 不应被“后台记录模式”影响。

## 失败排查
- 若两档差异不明显：
  - 检查是否真的切换成功（重进设置确认）。
  - 检查是否误开了 Journey。
  - 检查路线是否过短或全程开阔直线。
- 若高精度过于耗电：
  - 记录场景（高楼密集、弱信号、长时间屏亮）后再调整阈值。
