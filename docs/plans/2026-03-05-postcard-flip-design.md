# Postcard Flip Redesign 设计文档

## 目标
将明信片展示从“左右分栏”改为“正反面卡片”，并统一发送预览与收件箱详情体验：
- 点击卡片：翻面
- 长按卡片：保存当前面到相册

## 需求拆解
- 正面：上传图片为主体（尽可能大），叠加头像、城市定位、昵称
- 反面：message 为主体，底部信息为头像、城市定位、昵称
- 发送预览页与详情页视觉一致，避免两套样式漂移

## 方案选择
采用方案 1：抽取可复用 `FlippablePostcardView`。

### 核心设计
- 新增通用组件：
  - `PostcardFaceFrontView`
  - `PostcardFaceBackView`
  - `FlippablePostcardView`
- 组件输入统一为：图片来源、message、城市、昵称、是否正面。
- 翻面动效使用 3D Y 轴旋转。
- 长按回调由页面承接，页面负责保存当前面图片。

## 数据与兼容
- 发送接口仍只传一张图片：改为渲染“正面”图上传。
- 文案 message 仍通过原字段发送，不改变后端协议。
- 详情页若远端图加载失败，正面展示占位图，反面仍可正常浏览 message。

## 验收标准
- 预览页与详情页都能点击翻面。
- 预览页与详情页都能长按保存当前面。
- 正面图片视觉占比显著增大。
- 正反两面都包含头像/城市/昵称。
